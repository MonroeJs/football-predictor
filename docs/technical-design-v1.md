## 二、新增路由设计

### 2.1 新增 API 路由

```python
# 主页保持不变
@app.route('/')  # 已存在

# 新增 API
@app.route('/api/today')       # 今日比赛 + 推荐
@app.route('/api/calendar')    # 全部日历数据（按日期分组）
```

## 四、小组出线概率算法

采用 **Monte Carlo 模拟**：

```
对于每组4支队伍：
  1. 基于每场赔率计算 胜/平/负 概率
  2. 模拟 100,000 次完整小组赛
  3. 每次模拟：
     a. 按概率随机生成 6 场比赛结果
     b. 计算积分榜
     c. 排名（积分 → 净胜球 → 进球数）
     d. 记录前两名
  4. 统计各队出线频率 → 出线概率
  5. 同时统计小组头名概率
```

**实现细节**：
- 100,000 次模拟对 12 组 ≈ 1.2M 次，Python 约 1-2 秒
- 每次模拟结果缓存在内存中，赔率未变时不重算
- 同分按净胜球/进球数区分（模拟随机生成进球数）
- 进球数使用泊松分布（基于赔率隐含的期望进球）

## 五、前端页面布局

### 5.1 设计理念

**无 Tab，单页滚动 + 导航胶囊按钮切换内容区域**。
- 顶部 Hero 统计区（始终可见）
- 中部分段导航（胶囊按钮，点击切换主内容区）
- 主内容区（根据选择的导航显示不同内容）
- 右侧边栏始终固定（图表 + 分组概况 + 分级说明）

### 5.3 视觉风格

- **无 Tab 边框**：导航用胶囊按钮（pill nav），选中状态有发光效果
- **卡片圆角 16px**，半透明玻璃效果 (backdrop-filter: blur)
- **Hero 数字动画**：页面加载时从 0 数到目标值
- **响应式**：手机端右侧边栏折叠到底部
- **颜色**：延续暗色主题，导航胶囊用渐变背景标识选中状态

### 5.4 交互效果

- 导航切换时内容淡入淡出 (opacity transition 0.3s)
- 鼠标悬停表格行放大 1.02x
- 推荐投注卡片有绿色左边框脉冲动画
