# 世界杯 2026 预测系统 — 需求方案 v2

## 一、本轮目标

距离开赛还有 11 天，本轮聚焦 **开赛后的日常使用体验**。

## 二、功能需求

### F1. 开赛后追踪（最高优先）

**目标**：开赛后每天记录实际结果，对比预测准不准

**具体内容**：
- F1.1 每场比赛可录入实际比分（胜/平/负）
- F1.2 自动对比预测 vs 实际，统计累计准确率
- F1.3 按置信度分层展示实时准确率（High 多少、Elite 多少）
- F1.4 投注盈亏追踪：推荐了哪些、实际赢了多少

**验收标准**：
- 录入比分后自动刷新所有统计
- 准确率按天/累计两种视图
- 数据持久化（不刷新丢失）

### F2. 实时赔率更新

**目标**：每天自动拉取最新赔率，不用手动改 CSV

**具体内容**：
- F2.1 从 football-data.co.uk 自动下载最新 WC 赔率
- F2.2 合并到现有赔率文件，不覆盖已录入的结果数据
- F2.3 页面显示赔率更新时间

**验收标准**：
- 一键刷新赔率
- 赔率变动时显示差价提示

### F3. 预测历史面板

**目标**：查看过去几天的预测记录

**具体内容**：
- F3.1 历史预测列表：日期、对阵、预测、实际结果、是否命中
- F3.2 累计统计：总预测场次、命中率、各级别准确率
- F3.3 图表：每日准确率趋势

**验收标准**：
- 数据本地持久化（localStorage 或 SQLite）

### F4. 投注记录本（后续）

**目标**：记录实际投注和盈亏

**内容**：推迟到 v3，不影响开赛前使用

## 三、技术方案

- 数据存储：JSON 文件（`data/results/{date}.json`），简单可靠
- 结果录入：页面内嵌表单，点击比分即可
- 赔率更新：`scripts/fetch_live_odds.py`，手动或定时触发
- 准确率计算：后端聚合，前端展示

## 四、交付物

1. 结果录入界面（点击比赛 → 选结果）
2. 实时统计面板（准确率、盈亏）
3. 赔率更新脚本
4. 历史预测记录
